KYIVHELP — ДЕПЛОЙ НА VERCEL (покроково)
=========================================

Перед деплоєм підготуйте:
  • Акаунт GitHub з кодом проєкту
  • Проєкт Supabase (регіон Frankfurt рекомендовано)
  • Виконаний SQL з файлу supabase/schema.sql у Supabase SQL Editor


КРОК 1 — SUPABASE (база даних)
-------------------------------
1. Відкрийте https://supabase.com → увійдіть → New project
2. Оберіть регіон (Frankfurt) → задайте пароль БД → Create
3. Ліве меню: SQL Editor → New query
4. Відкрийте файл supabase/schema.sql у проєкті → скопіюйте весь текст → Run
5. Ліве меню: Project Settings → API
6. Скопіюйте:
     Project URL          → NEXT_PUBLIC_SUPABASE_URL
     anon public          → NEXT_PUBLIC_SUPABASE_ANON_KEY
     service_role         → SUPABASE_SERVICE_ROLE_KEY  (НЕ публікуйте в клієнті!)
7. Згенеруйте випадковий рядок 32+ символів → REVEAL_SECRET
   (можна: https://generate-secret.vercel.app/32 або PowerShell:
    [Convert]::ToBase64String((1..32|ForEach-Object{Get-Random -Max 256})))


КРОК 2 — GITHUB (якщо ще не завантажено)
-----------------------------------------
1. Створіть репозиторій на github.com (наприклад kiev-help)
2. У папці проєкту:
     git init
     git add .
     git commit -m "KYIVHELP initial deploy"
     git branch -M main
     git remote add origin https://github.com/ВАШ_ЛОГІН/kiev-help.git
     git push -u origin main


КРОК 3 — VERCEL: IMPORT PROJECT
---------------------------------
1. Відкрийте https://vercel.com → Log in (через GitHub)
2. Натисніть "Add New..." → "Project"
3. У списку репозиторіїв знайдіть kiev-help → натисніть "Import"
4. Configure Project:
     Framework Preset: Next.js (має визначитися автоматично)
     Root Directory: ./  (за замовчуванням)
     Build Command: npm run build
     Output Directory: .next  (за замовчуванням)


КРОК 4 — VERCEL: ENVIRONMENT VARIABLES
---------------------------------------
На екрані Configure Project (ПЕРЕД Deploy) або після:
  Settings → Environment Variables

Додайте кожну змінну (Name → Value → Environment: Production, Preview, Development):

  Name: NEXT_PUBLIC_SUPABASE_URL
  Value: (ваш Project URL з Supabase)

  Name: NEXT_PUBLIC_SUPABASE_ANON_KEY
  Value: (anon public key з Supabase)

  Name: SUPABASE_SERVICE_ROLE_KEY
  Value: (service_role key з Supabase)

  Name: REVEAL_SECRET
  Value: (ваш випадковий рядок 32+ символів)

  Name: NEXT_PUBLIC_SITE_URL
  Value: спочатку залиште порожнім або вкажіть https://ваш-проєкт.vercel.app
         (після першого деплою оновіть на фінальний URL домену)

Натисніть Save для кожної змінної.


КРОК 5 — DEPLOY
----------------
1. Натисніть "Deploy" (велика кнопка на Configure Project)
2. Дочекайтесь зеленого статусу "Ready" (1–3 хвилини)
3. Натисніть "Visit" — відкриється ваш сайт


КРОК 6 — ПІСЛЯ ДЕПЛОЮ (перевірка)
----------------------------------
1. Відкрийте https://ВАШ-ДОМЕН.vercel.app
2. Перевірте головну сторінку та /new
3. Перевірте API: https://ВАШ-ДОМЕН.vercel.app/api/health
   Очікується: {"status":"ok","supabase":true,"data":[]}
4. Якщо supabase: false — перевірте Environment Variables на Vercel
5. Створіть тестове оголошення на /new — воно має з'явитися у стрічці


КРОК 7 — ВЛАСНИЙ ДОМЕН (опційно)
---------------------------------
1. Vercel → ваш проєкт → Settings → Domains
2. Add → введіть домен → дотримуйтесь інструкцій DNS
3. Оновіть NEXT_PUBLIC_SITE_URL на новий домен → Redeploy


ЛОКАЛЬНА РОЗРОБКА
------------------
1. Скопіюйте .env.example → .env.local
2. Заповніть усі значення ключів
3. npm install
4. npm run dev
5. Відкрийте http://localhost:3000
